1. >Huaban<  花瓣网爬取

原理
  1. 直接在图片上右击后选择复制图片链接，是以https://hbimg.huabanimg.com/开头的，而花瓣网所有图片的真实地址是保存在http://hbimg.b0.upaiyun.com/中的。我瞎猜一下，这个地址大概是临时的，类似于Windows中的快捷方式，在爬取的时候，应当把前缀改成云盘地址。
  2. 有机所得图片地址的后面一般以_fw658/format/webp结尾，这段东西不需要。
  3. 图片链接地址去掉上述首尾，就剩下一串乱七八糟的字符串，这就是花瓣网上每一张图片的身份证。用身份证代码加上云盘前缀，就是图片的真实地址。
方法

  • .run(ask, lenth=1000) # 获取画板下所有图片的链接集合

    ask :: 画板名字（依据文件中的设定）

    lenth :: 获取画板中前n张图，默认1000张

    返回值 :: 所有图片地址组成的列表

  • .download(lenth=1000) # 根据依赖文件下载图片

    lenth :: 下载图片数量上限，默认1000

    依赖文件 :: picurls.txt

  • .get_source(a) # 根据依赖文件获取画板的URL链接

    a :: 画板名字（依据文件中的设定）

    依赖文件 :: picSpace.xml

采坑记录

    1. 爬取图片不完整

    原因：爬取时，是游客身份，部分图片无法获取（这些图片只有用户本人可见）

    解决：

    (1) 打开花瓣网网页，退出登录。
    (2) F21打开浏览器开发者模式，点开【network】，勾选【Preserve log】,清空监控。
    (3) 点击网页的登录按钮并登录花瓣网账号，目光来到右侧【Network】界面，找到【auth/】，这就是账号登录的post请求，点开后往下翻能够在【Form Data】看到账号、密码
    (4) 找到【auth/】下面的【huaban.com】，这是登录后重新加载的花瓣网界面，点开她，复制【Cookie】，把Cookie粘贴到爬虫代码的【header】字典中，带上header爬取，就能爬到原来丢失那部分图片。
